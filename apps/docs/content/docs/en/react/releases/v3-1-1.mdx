## What's New

### Tooltip Delay CSS Variables

[Tooltip](/docs/components/tooltip) now supports global delay configuration through theme variables ([#6617](https://github.com/heroui-inc/heroui/pull/6617)):

- `--tooltip-delay` — delay before showing a tooltip (default: `1500ms`)
- `--tooltip-close-delay` — delay before hiding a tooltip (default: `500ms`)

Override them globally:

```css
:root {
  --tooltip-delay: 700ms;
  --tooltip-close-delay: 0ms;
}
```

Individual `delay` and `closeDelay` props still override these values on specific tooltips.

<Callout type="warning">
  **Behavior change:** When using the default HeroUI theme, tooltip delays now default to `1500ms` / `500ms` instead of the previous React Aria defaults of `700ms` / `0ms`. Set the CSS variables or props explicitly if you want to preserve the old timing.
</Callout>
